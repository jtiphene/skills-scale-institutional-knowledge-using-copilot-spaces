# QA Checklist (Template)

This checklist is maintained by the QA Lead to ensure consistent verification of features.

Before QA run
- [ ] Acceptance criteria are attached to each story
- [ ] Test environments available and seeded with representative data
- [ ] Test data and test accounts documented

Functional testing
- [ ] All functional acceptance tests executed
- [ ] Regression smoke tests executed for impacted flows
- [ ] Cross-browser checks (if applicable)
- [ ] Accessibility quick checks for impacted UIs

Non-functional testing
- [ ] Performance smoke checks for critical flows
- [ ] Security scan validations (or note security exceptions)

Sign-off
- [ ] QA Lead confirms defects are at acceptable levels for release
- [ ] QA provides a test summary and any observed risks in the release ticket

Notes
- Link specific automated test runs or CI artifacts when available.
